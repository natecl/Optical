## Phase 5: Advanced Features
- [ ] Build visual processing module in `lib/vision` for extracting objects from video frames.
- [ ] Add frame sampling pipeline (target 1-2 FPS for inference) with backpressure protection to avoid request pileups.
- [ ] Define typed detection schema (id, label, confidence, bounding box, timestamp) shared across frontend/backend.
- [ ] Add frame pre-processing utilities (resize, compress, normalize) to reduce payload size before inference.
- [ ] Add object tracking between frames (stable IDs + smoothing) to reduce overlay jitter.
- [ ] Implement confidence thresholding and de-duplication for overlapping detections.
- [ ] Implement `OverlayLabels` for on-screen object identification.
- [ ] Render boxes/labels with viewport-safe positioning and collision handling for label readability.
- [ ] Add responsive overlay scaling for mobile/desktop and orientation changes.
- [ ] Add accessibility fallbacks (text list of detected objects when overlays are hidden/unsupported).
- [ ] Add `ModeToggle` (e.g., Learning Mode vs Standard Mode).
- [ ] Define explicit mode behavior matrix:
- [ ] `standard`: concise direct answers
- [ ] `learning`: step-by-step explanations + follow-up prompts
- [ ] `quiz`: questions first, reveal explanation after user attempt
- [ ] Persist mode in client state and include mode in all `interact`/`chat` payloads.
- [ ] Add telemetry hooks for advanced features (detection latency, overlay render time, mode usage).
- [ ] Acceptance criteria for Phase 5:
- [ ] Overlay updates smoothly under active camera stream without blocking UI interactions.
- [ ] Mode switch changes backend response style within one round-trip.
- [ ] Detection metadata is visible in transcript/debug logs for verification.
